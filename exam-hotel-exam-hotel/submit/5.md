# Front 구현

## 문제점

```
Session 기반의 인증 문제점
session 기반의 인증은 서버측에서 세션 정보를 관리하므로 로드벨런싱으로 인해 이중화가 되어 있을 경우 한쪽의 프론트 서버에 생성된 세션이 이중화로 인해
다른쪽 프론트 서버에는 세션이 존재하지 않게 된다. 세션 정보가 없는 프론트 서버로 사용자 요청이 발생한 경우 사용자의 요청이 정상적으로 처리되지 않을 수
있다는 문제점이 있다.
```

## 해결방법

```
1. Redis와 같은 데이터베이스를 사용하여 세션 정보를 저장하고 이중화된 프론트 서버간에 세션 정보를 공유할 수 있도록 구성한다.

2. JWT기반의 인증은 서버에서 세션 정보를 저장하지 않고, 클라이언트가 각각의 요청 헤더에 JWT 정보를 포함하여 이용되기 때문에 프론트 서버가 이중화 되어 
있더라도 이중화된 서버에서 세센이 불일치하는 경우와 같은 상황이 발생하지 않기 때문에 JWT 기반의 인증 방식을 사용한다. 
```
